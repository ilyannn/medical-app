# Security Policy

This project is designed to be open source while remaining safe for public collaboration.

## Supported usage

- Synthetic fixtures and fake integrations are safe to share publicly.
- Real household medical data, machine-specific paths, and private service credentials are not.

## Reporting a vulnerability

Please open a private security advisory or contact the maintainers directly if you discover a vulnerability that could expose user data, local filesystem access, or privileged bridge behavior.

## Sensitive areas

- Local document root management
- Contacts and calendar bridge behavior
- Paperless import credentials
- AI provider credentials
- MCP mutation tools
